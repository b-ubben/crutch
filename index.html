Crutch Typography Layout Forms and Input Utilities

This is Crutch.

Crutch is a lightweight and clean CSS framework offering straightforward, responsive front-end development in a smaller and easier to understand package than other similar frameworks. Crutch uses a simple three-column grid system for layout and comes with a host of utilities for margin, padding, positioning, and flexbox classes as well as default styling for many HTML elements. Styling is normalized for cross-browser rendering consistency and many elements are easily manipulated and customized via SCSS/SASS variables. Designed for ease-of-use and simple deployment for web-based projects focused more on function than form by developers who don't necessarily have the skill or the need for extensive hand-coded CSS styling and layout. Much of the initial SCSS/SASS file structure and implementation was heavily based upon Boostrap 4, however not much remains of Bootstrap, save for the inspiration.

This manual is constantly being updated as the framework grows.

Layout, Positioning, Navigation and Panes

Crutch uses a simple three column grid system for layout. Simply create a parent container called a .row, and within that row use either .item, .item-half, or .item-third to section off content. At mobile resolutions these items stack upon one-another. To keep these items from doing so, simply add the class .flex-nowrap to the parent row container element.

Included along with layout classes are a few classes to control positiong. These classes are:
.fixed-top
.fixed-bottom
.fixed-left
.float-left
.float-right
.clear (to clear content adjacent to floated elements)

Classes for creating containers of set width are present within the Crutch framework. These classes are: .container(which spans the whole width of the screen), .container-mobile, .container-tablet, .container-desktop, and .container-desktop-wide

Basic and effective navigation can be created with classes provided with Crutch. The basic setup of navigation such as the one pictured below has a container element with .navigation, and the links contained within have .navigation-link applied to them. The menu icon is generated by CSS, created by applying .menu-logo to a container element and .menu-logo-bars to empty divs or spans within that container. Any links contained within the collapsed menu below are contained in .nav-expanded container. Specialty text-styling for a company, brand, or website name can be applied with .navigation-logo and fixed navigation uses the .fixed-navigation class.

Menu interaction is acheived by including the crutch.js script file and applying the #menu to the menu logo's container element and #nav-expanded, .nav-expanded to the container of the links in the menu.

Page content may also be placed in a pane for visibility or organizational reasons. These panes included in Crutch make up most of the layout for this entire manual, in fact. Panes are created by a container element with class .pane and their contents with class .pane-content or .pane-title for complimentary styling. Panes may also be rounded by .pane-rounded and their colors and sizes may be manipulated by the utilities offered by Crutch. Panes are transparent grey by default and titles have the main theme color attributed to them.

Forms and Input

Crutch includes some minimal styling for form content that can be applied via .crutch-form. Inputs with type such as text, password and submit have default styling when .crutch-form is applied, and their width may be manipulated through either .input-long or .button-long classes. Two alert styling classes have been added to address the needs of those developers seeking to provide form submission feedback to their users. These classes are .alert-succesful and .alert-failure

Utilities

Crutch's main benefits are the utilites it offers for developers to manipulate the way that content is rendered on-screen.

Fluid image size for responsive design is provided by .image-fluid

Classes for visibility include: .visible, .invisible, and .screenreader-only

Utilities for margin and padding are essential, and have been included as the classes in the format of margin or padding followed by either just the amount (as either nothing, something, enough, more or most) for all around, or the position (as either top, bottom, left or right). Margin and padding along the X and Y axes are also supported (e.g. margin-x-something padding-y-something). Auto classes are also provided (e.g. margin-auto margin-top-auto margin-y-auto, etc.).
